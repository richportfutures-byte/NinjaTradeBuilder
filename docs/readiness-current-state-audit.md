# Readiness Current State Audit

## 1. Verified current readiness files and responsibilities
- [`pipeline.py`](../src/ninjatradebuilder/pipeline.py#L8-L19) and [`pipeline.py`](../src/ninjatradebuilder/pipeline.py#L70-L163) run the Stage A-D pipeline through prompt IDs 2-9 and do not import or call `run_readiness()`.
- [`runtime.py`](../src/ninjatradebuilder/runtime.py#L31-L37) registers the `readiness_engine_output` boundary, and [`runtime.py`](../src/ninjatradebuilder/runtime.py#L271-L299) owns `run_readiness()`, including trigger normalization, watchman-context injection, prompt 10 execution, and readiness-boundary enforcement.
- [`prompt_assets.py`](../src/ninjatradebuilder/prompt_assets.py#L9-L13) defines readiness constants, [`prompt_assets.py`](../src/ninjatradebuilder/prompt_assets.py#L493-L533) defines the standalone readiness prompt, and [`prompt_assets.py`](../src/ninjatradebuilder/prompt_assets.py#L536-L554) registers the shared-scope readiness prompt asset.
- [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L50-L96) defines readiness status, gate, reason, and trigger vocabularies, and [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L148-L324) defines `ReadinessDoctrineGateResult`, `ReadinessTriggerData`, and `ReadinessEngineOutput` plus readiness non-interchangeability validation.
- [`watchman.py`](../src/ninjatradebuilder/watchman.py#L52-L80) defines required runtime slots and readiness-context shape, [`watchman.py`](../src/ninjatradebuilder/watchman.py#L84-L248) builds deterministic readiness context from runtime inputs plus trigger data, [`watchman.py`](../src/ninjatradebuilder/watchman.py#L304-L323) validates trigger payloads, and [`watchman.py`](../src/ninjatradebuilder/watchman.py#L432-L569) classifies trigger state, event risk, and governance lockouts.
- [`readiness_adapter.py`](../src/ninjatradebuilder/readiness_adapter.py#L9-L30) converts a validated packet into readiness runtime inputs and currently supports packet-backed readiness for `ES`, `NQ`, `CL`, `ZN`, `6E`, and `MGC`.
- [`readiness_verify.py`](../src/ninjatradebuilder/readiness_verify.py#L43-L115) defines the readiness verification CLI surface, [`readiness_verify.py`](../src/ninjatradebuilder/readiness_verify.py#L182-L203) provides fixture-backed readiness inputs and triggers, and [`readiness_verify.py`](../src/ninjatradebuilder/readiness_verify.py#L422-L487) executes `run_readiness()` and writes operator-facing verification artifacts.
- [`readiness_web.py`](../src/ninjatradebuilder/readiness_web.py#L18-L132) defines the local readiness HTML surface, and [`readiness_web.py`](../src/ninjatradebuilder/readiness_web.py#L186-L256) serves the WSGI readiness app and local server entrypoint.

## 2. Exact current readiness vocabulary and statuses
- Top-level readiness status vocabulary is exactly `READY`, `WAIT_FOR_TRIGGER`, `LOCKED_OUT`, and `INSUFFICIENT_DATA`. [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L50-L55)
- Readiness output identity is `$schema = readiness_engine_output_v1`, `stage = readiness_engine`, and `authority = ESCALATE_ONLY`. [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L181-L198)
- Readiness doctrine gates are exactly `data_sufficiency_gate`, `context_alignment_gate`, `structure_quality_gate`, `trigger_gate`, `risk_window_gate`, and `lockout_gate`, and doctrine gate states are exactly `PASS`, `FAIL`, and `WAIT`. [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L56-L96)
- Wait reasons are exactly `entry_not_yet_confirmed`, `pullback_not_complete`, `breakout_not_confirmed`, and `timing_window_not_open`; lockout reasons are exactly `event_lockout_active`, `session_closed`, `risk_hard_stop_active`, and `governance_lock_active`; insufficient-data reasons are exactly `missing_required_fields`, `stale_market_packet`, `incomplete_contract_extension`, `invalid_challenge_state`, and `missing_trigger_context`. [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L65-L83)
- Watchman trigger-context states are exactly `scheduled_recheck_pending`, `scheduled_recheck_due`, `above_trigger_level`, `below_trigger_level`, and `at_trigger_level`. [`watchman.py`](../src/ninjatradebuilder/watchman.py#L36-L42)

## 3. Supported trigger families
- Current readiness trigger families are exactly `recheck_at_time` and `price_level_touch` in readiness constants, runtime normalization, schema validation, and watchman trigger validation. [`prompt_assets.py`](../src/ninjatradebuilder/prompt_assets.py#L9-L13) [`runtime.py`](../src/ninjatradebuilder/runtime.py#L174-L217) [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L84-L87) [`watchman.py`](../src/ninjatradebuilder/watchman.py#L304-L323)
- `recheck_at_time` requires `recheck_at_time` and forbids `price_level`, while `price_level_touch` requires `price_level` and forbids `recheck_at_time`. [`schemas/outputs.py`](../src/ninjatradebuilder/schemas/outputs.py#L154-L178)
- Within the inspected readiness files, supported trigger families are limited to `recheck_at_time` and `price_level_touch`; no inspected readiness file adds support for `time_recheck`, `price_level_break`, or `event_window_clear`. [`prompt_assets.py`](../src/ninjatradebuilder/prompt_assets.py#L518-L519) [`runtime.py`](../src/ninjatradebuilder/runtime.py#L185-L217) [`watchman.py`](../src/ninjatradebuilder/watchman.py#L304-L323)

## 4. Whether a readiness web surface exists, and where
- Yes. [`readiness_web.py`](../src/ninjatradebuilder/readiness_web.py#L186-L256) defines `build_readiness_web_app()` and the local readiness server entrypoint.
- The web surface serves a local HTML form titled `Readiness Engine` with packet, runtime-input, and trigger fields. [`readiness_web.py`](../src/ninjatradebuilder/readiness_web.py#L18-L132)
- The web API exposes `POST /api/readiness`, accepts exactly one of `packet` or `runtime_inputs` plus `readiness_trigger`, converts packets through `build_readiness_runtime_inputs_from_packet()`, and then calls `run_readiness()`. [`readiness_web.py`](../src/ninjatradebuilder/readiness_web.py#L201-L238)
